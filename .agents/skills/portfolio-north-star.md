# Skill: Portfolio North Star

## Purpose
Use this when working on the portfolio app's UX, visual direction, copy hierarchy, or overall page composition.

## Outcome Standard
The site should make a technically strong visitor think:
- this person can build
- this person has taste
- this person understands product presentation

## Core Positioning
Mahfuzur Rahman should be presented as:
- software engineer
- full-stack builder
- systems-minded operator
- someone who can bridge backend depth and polished frontend delivery

## Desired Brand Feel
- premium
- modern
- cinematic restraint
- technically credible
- calm confidence

## Recommended Visual Blend
Mix these references intentionally:
- **Linear** -> hierarchy, spacing, product precision
- **Framer** -> visual drama, hero framing, motion taste
- **Vercel** -> minimal confidence, typography discipline

Do not copy any one source blindly.

## Home Page Requirements
The home page should eventually contain:
1. hero with clear value proposition
2. featured project proof
3. strengths / capabilities section
4. project spotlight grid
5. credibility signals
6. blog / writing teaser
7. contact or CTA section

## Copy Rules
- short, sharp, specific
- avoid generic buzzwords
- no fake-founder fluff
- show outcomes and capabilities, not empty adjectives

## Design Rules
- one strong idea per section
- generous spacing
- clear section rhythm
- contrast through typography and surface layering
- motion used sparingly and purposefully

## Anti-Patterns
Avoid:
- crowded hero sections
- random gradients everywhere
- too many card styles
- more than 1-2 accent colors without reason
- weak typography hierarchy
- animation that delays reading

## Review Checklist
Before accepting a design direction, ask:
- does this feel memorable?
- does this feel expensive?
- does this still feel usable?
- does this make Mahfuz look credible, not cosplay-creative?
- would another engineer or recruiter remember it tomorrow?
